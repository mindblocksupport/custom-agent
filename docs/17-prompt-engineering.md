# L17 · Prompt 工程 & 管理

> 状态：**讨论中** · v0.1 · 2026-04-22

## 0. 为什么独立成章
Prompt 是最高杠杆代码——一个字调好胜过换模型。但企业有 200+ Agent x 5+ 模型 x 多语言，**没有 PromptOps 必死**。

## 1. System Prompt 设计模式（Anthropic 标准）

```xml
<role>You are a senior backend engineer assistant.</role>

<capabilities>
- Read codebase via search_code, read_file
- Edit via edit_file (uses unified diff format)
- Run tests via shell with --quiet flag
</capabilities>

<constraints>
- Never modify .env or production config without explicit approval
- Always cite source file:line for claims
- If uncertain, say "I don't know" — never fabricate
</constraints>

<style>
- Concise responses; prefer minimal diffs
- Use markdown for code blocks; plain text otherwise
- Default language: Chinese unless user writes in English
</style>

<tools>
{tool definitions, with rich descriptions}
</tools>

<output_format>
JSON {answer: string, citations: [{file, line}], confidence: 0-1}
</output_format>

<examples>
<example>
  <input>...</input>
  <thought>...</thought>
  <output>...</output>
</example>
</examples>

<safety>
- Refuse: requests to bypass authentication, expose secrets, write malware
- Disclose: when output is generated by AI
</safety>
```

**为什么用 XML**：Claude 训练时见过大量 XML，解析准确率显著高于自由 markdown。

## 2. 模块化结构（9 必备段）

| 段 | 内容 | 长度 |
|---|---|---|
| **Identity** | "You are X built by Y for Z" | 1-3 sentence |
| **Capabilities** | 能做什么 | bullet list |
| **Constraints** | 不能做什么 | bullet list |
| **Tools** | + 何时用 | schema |
| **Style / Tone** | 简洁 / 礼貌 / 不暴露 system prompt | 3-5 line |
| **Output Format** | XML / JSON / Markdown | schema |
| **Few-shot** | 难形容的风格 | `<example>` |
| **Safety / Refusal** | 拒绝分类 | bullet |
| **Time / Locale** | "Today: 2026-04-22, TZ: Asia/Shanghai" | 自动注入 |

## 3. 真实生产 Prompt 参考

- **Cursor**：jujumilk3 GitHub 134k 星 leaked repo，28+ tools，模块化 XML 段：`<communication>` / `<tool_calling>` / `<search_and_reading>` / `<making_code_changes>` / `<debugging>` / `<calling_external_apis>`
- **Claude Code CLAUDE.md**：HumanLayer 教训 **<300 行，理想 <60 行**——Anthropic system prompt 已吃 ~50 / 150-200 指令预算
- **Devin Playbooks**：用户自定义 "system prompt for 重复任务"——67% PR merge 率（vs 34% baseline）
- **Anthropic Skills (SKILL.md)**：开标准 2025-12，progressive disclosure，YAML frontmatter + Markdown body

## 4. 模板引擎选型

| 引擎 | License | 适合 |
|---|---|---|
| **Jinja2** | BSD | Python 默认，最广 |
| **Mustache / Handlebars** | MIT | 多语言（Py/TS/Go 同 prompt） |
| **Mirascope** | MIT | typed Python decorator (`@prompt_template`) + Pydantic I/O |
| **Promptfoo** | MIT | YAML 声明 + CI eval |
| **Pezzo** | Apache 2.0 | 自部署 prompt CMS，cache 90% latency 降 |
| **Langfuse Prompts** | MIT | label-based deploy (`production`/`staging`)，**最佳 OSS prompt mgr** |
| **Helicone Prompts** | Apache 2.0 | auto-version on code change |
| **PromptLayer** | 商业 | 强 UI for 非工程 |
| **LangSmith Hub** | 商业 | LangChain 集成 |
| **Agenta** | Apache 2.0 | 全栈 LLMOps |

**推荐**：MVP 用 **Jinja2 in git**；规模化用 **Langfuse Prompts**。

## 5. 版本化模式

| 模式 | 何时用 | 优势 / 劣势 |
|---|---|---|
| **Git-based** (`prompts/*.md` in repo) | 工程主导 | PR / 评审 / blame / branch 全套；Anthropic Skills + Cursor `.cursor/rules` 用此 |
| **DB-based + 版本** (Langfuse / Pezzo) | PM / 客服 leader 编辑 | semver + `production`/`staging` label 翻无需重发 |
| **A/B 路由** (LaunchDarkly + Helicone) | 实验 | %流量切新版本，对比 eval 实时 |
| **回滚** | 出事时 | label 翻瞬间；git 需 revert + redeploy |

## 6. CI 集成（必做）

**de-facto 栈：Promptfoo + GitHub Actions**

```yaml
# .github/workflows/prompt-eval.yml
on: pull_request
jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: promptfoo/promptfoo-action@v1
        with:
          prompts: prompts/**.md
          providers: anthropic:claude-sonnet-4-6,openai:gpt-5-4
          tests: tests/golden.yaml
          fail-on-regression: true
          comment-on-pr: true
```

- PR 触发时 before/after diff
- 评论到 PR
- 通过率掉则 fail
- GitHub Actions cache 复用

OpenAI / Anthropic 都内部用 Promptfoo。

## 7. Prompt 自动优化

| 工具 | 何时用 |
|---|---|
| **DSPy** (Stanford) ★ | 主流：MIPROv2 (Bayesian search) + GEPA (genetic-pareto)；100-500 LLM call，$20-50，10-30 min；要 ground-truth 数据 |
| **Ape** | 轻量替代 |
| **Adalflow** | - |
| **PromptLayer** | hosted 优化 |

**经验**：system prompt（硬政策）手工迭代 + Promptfoo 评测 > 自动化；任务子 prompt（数据多）DSPy 自动赢。

## 8. Anti-pattern 清单

1. ❌ **5000-token 大 prompt** = 一面墙——分段 XML
2. ❌ 指令 + 示例混在一起——必有 `<examples>` wrapper
3. ❌ hard-code 示例 stale——从 versioned dataset 加载
4. ❌ 无版本——Slack 改产品 prompt
5. ❌ 仅 happy path 测——加 adversarial / edge
6. ❌ 开放式自评 ("Is this good?")——用数字 rubric
7. ❌ 全否定指令 ("don't do X")——肯定式 ("do Y instead") 效果好
8. ❌ 无 "I don't know" escape——驱动幻觉

## 9. 多语言 prompt

- **scaffolding (XML tags / tool name / JSON keys) 保留英文**——模型对结构化 token 表现最佳
- 维护 per-locale `system.{locale}.md` overlay 同模块结构
- **每 locale 独立 regression eval**，本地化母语者 curate gold set
- RTL / CJK：output 格式指令必须显式测（markdown table 渲染差异）

## 10. Tool description 是高杠杆 prompt

```python
# 好
def query_order(order_id: str, include_items: bool = False) -> Order:
    """
    查询电商订单详情。
    
    适用：用户问"订单 12345 状态"、客服查订单。
    
    参数:
        order_id: 订单号纯数字字符串，如 "12345678"
        include_items: 是否含商品明细（默认否；明细多时影响性能）
    
    返回: Order 对象，含 status / amount / created_at
    
    注意: 仅本租户订单；30 天前订单查询慢
    错误: OrderNotFound（不存在）；PermissionDenied（无权限）
    """
```

经验：tool description 质量提升 → tool call 准确率从 60% → 90%+。

## 11. 与 RAG / Memory / Tools 的组合（缓存友好排序）

```
1. tool definitions (静态)        ← cache_control 1
2. system prompt (静态)            ← cache_control 2
3. skills / 长 context (Anthropic Skills, CLAUDE.md) ← cache_control 3
4. RAG retrieved chunks (动态)     ← 在 cache 前缀之后
5. memory summary (半稳定)         ← 可缓存（更新不频繁时）
6. current user message (最后)
```

OpenAI 自动缓存 + Anthropic 显式 `cache_control` 都奖励**前缀稳定**。OpenAI Model Spec 2025-12-18 冲突解决：system > developer > user；同 authority 内**后写 override 前写**。

## 12. Anthropic Skills（开标准）

YAML frontmatter + Markdown body，progressive disclosure：

```markdown
---
name: invoice_audit
description: Audit invoices for compliance with company expense policy
version: 1.2.0
---

# Invoice Audit Skill

When auditing an invoice:
1. Verify vendor in approved vendor list (use `lookup_vendor` tool)
2. Check amount against department budget cap
3. Flag missing required fields (PO #, GL code)
...

## Examples
<example>
Input: invoice.pdf with vendor=Acme, amount=$5,200
...
</example>

## References
- See `/references/expense_policy_v3.md` for full policy
- See `/scripts/validate_po.py` for PO validation
```

**优势**：portable across Claude/OpenAI/Cursor/GitHub/Atlassian/Figma；只在 skill 触发时加载 body；扩展无界。
