# L32 · 联邦知识 / 跨租户共享

> 状态：**讨论中** · v0.1 · 2026-04-22 · 前瞻 / 行业 Agent 杠杆

## 0. 场景驱动
- 行业内通用知识（法规 / 标准 / 公开案例）跨企业重用
- 联盟 / 协会 / 供应链共享
- 但**单租户私有数据绝不混淆**

## 1. 知识层级

```
┌──────────────────────────────────────┐
│ Public 公共层                         │
│  - 法规 / 标准 / 公开案例              │
│  - 行业百科                           │
│  - 我们 curated                       │
├──────────────────────────────────────┤
│ Industry / Federated 行业联邦层      │
│  - 联盟成员同意共享                    │
│  - 去标识化                           │
│  - 治理 / governance 委员会           │
├──────────────────────────────────────┤
│ Tenant 私有层                         │
│  - 客户专有                           │
│  - 严格隔离 (L24)                     │
└──────────────────────────────────────┘
```

## 2. 共享技术（按隐私强度）

| 技术 | 隐私 | 实用 |
|---|---|---|
| **数据共享 + 协议**（明文） | 弱 | 高 |
| **去标识化共享** | 中 | 高 |
| **K-anonymity / 差分隐私** | 强 | 中 |
| **联邦学习** (FL) | 强 | 中 (训练场景) |
| **Secure Multi-party Computation** (MPC) | 极强 | 低 (慢) |
| **Homomorphic Encryption** (HE) | 极强 | 低 (慢) |
| **Confidential Computing** (TDX/SEV) | 强 | 中高 |

## 3. 实用模式

### 3.1 Federated Knowledge Base
- 行业法规 / 标准统一维护
- 我们提供 curated KB
- 客户付订阅费，按需调用

### 3.2 联邦 RAG
- Query 在客户私有 KB + 联邦 KB 都查
- 答案融合
- 引用清晰来源

### 3.3 联邦 Skill 市场
- 客户上架 skill 给同行业
- 收益分成
- 安全审核 / 治理

### 3.4 联邦学习（高级）
- 多客户参与训练
- 数据不出本地
- 模型参数聚合（FedAvg / FedProx）
- 适合：行业大模型微调

## 4. 治理结构

跨企业知识联邦必须有：
- **治理委员会**（成员代表）
- **数据共享协议**（DSA）
- **质量标准**
- **争议解决**
- **退出机制**

## 5. 真实案例

- **金融风控联邦**（央行试点）：跨银行黑名单
- **医疗影像联邦**：跨医院模型训练
- **供应链 trace**：上下游协同（区块链 + AI）
- **欧洲 Gaia-X**：主权数据空间

## 6. 法律 / 合规

- 跨境联邦：数据出境合规（PIPL）
- 反垄断：避免共享商业敏感（价格 / 客户）
- 个人信息保护：去标识化必须

## 7. 我们的策略

**短期不做**，但留:
- KB 层级设计（public / federated / tenant）
- Skill registry 支持 visibility 标签
- 长期：行业 vertical 成熟后启动联邦
