# L25 · 数据合规 & 数据出境

> 状态：**讨论中** · v0.1 · 2026-04-22 · 政企 RFP 第一关 / 国际客户 GDPR 必备

## 0. 为什么独立成章
- 国内：个保法 + 数据安全法 + 网络安全法 + 大模型备案 + 等保
- 国际：GDPR + CCPA + SOC2 + ISO27001 + EU AI Act
- 行业：金融（人行/银保监）+ 医疗（HIPAA）+ 政务（信创）
- **任何一项不达标 = deal 死**

## 1. 国内合规矩阵

| 法规 | 适用 | 关键要求 |
|---|---|---|
| **网络安全法** | 全部 | 等保备案、关基保护 |
| **数据安全法** | 全部 | 数据分类分级、跨境评估 |
| **个人信息保护法 (PIPL)** | 涉及 PI | 知情同意、最小必要、可删除、单独同意敏感 PI |
| **生成式 AI 服务管理办法** (2023-08) | 大模型对外服务 | 算法备案、训练数据合规、安全评估 |
| **大模型备案** (网信办) | 提供 LLM 服务 | 安全评估 + 备案号 |
| **算法推荐管理规定** | 推荐 / 个性化 | 算法备案 |
| **GB/T 45654-2025** (生成式 AI 安全基本要求) | 国产 LLM | 完整安全要求 |
| **GB/T 45652-2025** (训练数据安全) | 训练数据 | 数据来源 / 标注 / 风险 |
| **等保 2.0** | 系统分级 | 三级要求 ≈ 200 项 |

### 必须做的（最小集）
1. ✅ 网信办**大模型备案**（提供 LLM 服务必须）
2. ✅ **算法备案**（推荐 / 个性化）
3. ✅ **等保三级**（核心业务）
4. ✅ **PIPL 合规**：知情同意 + 最小必要 + 可删除
5. ✅ **数据分类分级**（核心 / 重要 / 一般）
6. ✅ **跨境数据评估**（涉及境外服务时）
7. ✅ **AI 输出标识**（GB/T 45654 要求）
8. ✅ **敏感词 + 内容安全审核**（按 GB/T 45654）
9. ✅ **训练数据 audit trail**（GB/T 45652）

## 3. 数据分类分级（PIPL + DSL）

```
┌─────────────────────────────────────────────────┐
│ 核心数据 (国家安全 / 国民经济)                  │
│  → 严格管控，不出境                              │
├─────────────────────────────────────────────────┤
│ 重要数据 (企业敏感 / 大量 PI)                   │
│  → 出境必经评估                                  │
├─────────────────────────────────────────────────┤
│ 一般数据 (公开 / 普通业务)                       │
│  → 一般管理                                      │
├─────────────────────────────────────────────────┤
│ 个人信息                                          │
│  ├─ 一般 PI: 知情同意                            │
│  └─ 敏感 PI (生物 / 医疗 / 金融 / <14岁):        │
│      单独同意 + 必要性 + 安全保护                 │
└─────────────────────────────────────────────────┘
```

## 4. 数据出境路径（PIPL 第 38 条）

3 种合规路径（境外接收方处理 PI 必须选一）：

1. **国家网信部门安全评估** —— 处理大量 PI 或重要数据时强制
2. **专业机构个人信息保护认证**
3. **标准合同**（CAC 标准合同备案）

**实操**：
- 默认设计**不出境**（国内场景）
- 如客户允许跨境，走标准合同 + 评估
- 涉外 SaaS 必须**多 region**（中国 / 海外分别部署）

## 5. 数据不出域设计 (Sovereign AI)

国内政企 / 涉密项目硬要求：
- ✅ 模型**本地部署**（不能调海外 API）
- ✅ 向量库**本地部署**
- ✅ 日志**不出域**
- ✅ 严控**公网出口**（白名单）
- ✅ trace / observability **客户机房**
- ✅ Backup / 归档**域内**

技术栈：
- 模型：DeepSeek V3.2 / Qwen3-235B / GLM-4.6 / Doubao 自部署
- vLLM / SGLang 推理
- 私有 RAGFlow + Qdrant + Postgres
- 自部署 Langfuse

## 6. PIPL 实操清单

### 6.1 Privacy Policy 必含
- 收集 PI 类型 / 目的 / 方式
- 保存期限
- 接收方（含 LLM vendor）
- 用户权利（查 / 复 / 改 / 删 / 移植 / 撤销同意）
- 联系 DPO

### 6.2 用户权利接口（必须能技术实现）
- 查询：用户能看自己所有数据
- 删除：**级联删** DB + 向量 + 缓存 + log + Memory + 备份（GDPR right to be forgotten）
- 导出：JSON / CSV 全量
- 撤销同意：重启账户

### 6.3 同意管理
- 注册时 explicit consent
- 敏感 PI 单独同意
- 同意可撤回 + 后果说明

### 6.4 跨境同意
- 单独勾选 "我同意 PI 出境"
- 列明境外接收方 + 国家 + 用途

## 7. AI 特定合规（GB/T 45654 + EU AI Act）

### 7.1 输入输出审核
- 用户输入扫违法违规内容
- 输出扫不当内容
- 高风险拦截 + 人审

### 7.2 AI 输出标识
- 文本：可识别 marker
- 图像：watermark
- 视频 / 音频：标识

### 7.3 训练数据治理
- 来源合法
- 标注规范
- 风险数据剔除
- 完整 audit trail

### 7.4 EU AI Act 高风险义务
- 风险管理体系
- 数据治理
- 技术文档
- 透明度（用户知道在和 AI 对话）
- 人工监督
- 准确性 / 健壮性 / cybersecurity
- 合格评定

## 8. 隐私技术（PETs）

| 技术 | 用途 |
|---|---|
| **PII 检测 / 脱敏** (Microsoft Presidio) | 入参 / 出参 / log 自动脱敏 |
| **Differential Privacy** | 训练 / 统计加噪 |
| **Federated Learning** | 数据不出本地训练 |
| **Homomorphic Encryption** | 加密下计算（性能差，慎用） |
| **Confidential Computing** (Intel TDX / AMD SEV-SNP) | 内存加密 + 可信证明 |
| **Anonymization / Pseudonymization** | 去标识化 |
| **K-anonymity** | 统计去识别化 |

## 9. Vendor 处理协议（DPA）

签所有 vendor：
- LLM provider (OpenAI / Anthropic / 国内厂商)
- Cloud provider
- Observability (Langfuse / Datadog)
- Vector DB / DB SaaS

DPA 必含：
- 数据处理目的 / 期限
- 安全措施
- 子处理者列表
- breach 通报义务（72h GDPR）
- 数据返还 / 销毁

## 10. Breach 响应

```
检测 → 评估 → 控制 → 通知 → 报告 → 修复 → 复盘
```

时效要求：
- **GDPR**: 72h 内通知监管 + 受影响用户
- **PIPL**: 立即通知监管 + 受影响个人
- **HIPAA**: 60 天内
- **CCPA**: 立即
