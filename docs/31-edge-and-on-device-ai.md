## 3. 部署模式

### 3.1 端为主，云兜底
- 简单查询端跑（隐私 + 快）
- 复杂查询升级云

### 3.2 云为主，端缓存
- 默认云
- 网络差 / 离线时端兜底（受限能力）

### 3.3 端云协同（最强）
- 端：意图识别 / 简单回复 / 流式 STT
- 云：复杂推理 / 工具调用
- 隐私敏感数据**永不上云**

## 4. AutoGLM 模式（手机操作）

Zhipu **AutoGLM 2025-12 开源**，端 9B VLM：
- WeChat / Meituan / Didi UI 自动操作
- ADB 通道（Android 友好）
- 用户隐私不出端

可借鉴：移动端 Agent 不只是 chat，**能操作手机 app**。

## 6. 我们的策略

**不优先做端**，但**架构留口**：
- LiteLLM gateway 把"端模型"当一个 provider
- 端 SDK（Python / Swift / Kotlin / TS）通过统一 API 访问
- 知识 / Skill / Memory 云端管，端按需拉取
- 离线时降级模式

仅在客户场景驱动时投入：
- 政企涉密项目（必须端 + 离网）
- 车企 / 制造客户
